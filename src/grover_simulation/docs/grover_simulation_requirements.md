# Grover Simulation 要件定義書 (Requirements Definition)

## 1. プロジェクトの目的と背景
本プロジェクトは、Groverのアルゴリズムを用いた組合せ最適化問題（ナップサック問題、ルーティング問題など）の解法をシミュレーションし、理論的な量子優位性および現実的なノイズ環境下での性能限界を調査・評価することを目的とする。古典アルゴリズム（全探索やヒューリスティクス）との比較を通じて、NISQ時代および誤り耐性量子計算(FTQC)時代に向けたアルゴリズムの実用性を検証するツールキットを提供する。

## 2. シミュレーションの対象とスコープ
### 対象とする問題 (Problems)
- **ナップサック問題 (Knapsack Problem)**
- **ルーティング問題 (Routing/TSP-like Problem)**
- **スケジューリング問題 (Scheduling Problem)**
※共通のインターフェースを通じ、将来的に新たな問題（MAX-CUTなど）を容易に追加できること。

### スコープ
- オラクル回路の自動生成ロジックの実装
- 理想的なシミュレーション（Statevector）およびノイズありシミュレーション（QASM等）のサポート
- 古典アルゴリズムとの比較（計算ステップ数、正答確率、実行時間など）
- カスタムノイズモデル（ゲートエラー、読み出しエラー、熱・位相緩和など）の適用と検証

※実機（IBM Quantumハードウェアなど）での実行そのものは初期スコープ外とするが、実機を模擬したノイズモデルを重要視する。

## 3. 機能要件 (Functional Requirements)

### 3.1 問題定義 (Problem Definition)
- 各問題のインスタンス（サイズ、制約、コスト関数）を動的に生成できること。
- 問題を古典アルゴリズム・量子アルゴリズムで共通して評価できるインターフェースを持つこと。

### 3.2 量子シミュレーション (Quantum Simulation)
- 与えられた問題制約を評価し、解の位相を反転させるオラクル(Oracle)回路を動的に構築できること。
- オラクルと振幅増幅(Diffuser)を組み合わせ、Groverのメイン回路を構築できること。
- 任意の確率的エラー（Gate Error, Readout Error）および物理的ノイズを満たすノイズモデルを注入できること。
- シミュレーションを実行し、サンプリング結果(Counts)および成功確率を取得できること。

### 3.3 古典シミュレーション (Classical Simulation)
- 全探索(Brute Force)により、厳密な最適解と、Groverと比較可能な探索ステップ数を算出できること。
- （オプション）貪欲法などのヒューリスティックによる近似解を提供できること。

### 3.4 ベンチマーク・可視化 (Benchmark & Visualization)
- 量子と古典の実行結果を一元管理し、成功確率や必要イテレーション数などのメトリクスを算出できること。
- ノイズレベルの変動に対する成功確率の推移をグラフ(Matplotlib等)で可視化できること。
- 回路の深さ(Depth)や使用する量子ビット数など、回路の複雑さを出力できること。

## 4. 非機能要件 (Non-Functional Requirements)
- **拡張性**: 新しい組合せ問題や、新しいノイズモデルを最小限のコード修正（プラグイン的）で追加できるアーキテクチャであること。
- **可読性・保守性**: Pythonの型ヒント(Type Hints)を積極的に用い、Docstring (Googleスタイル推奨) で入出力とアルゴリズムの挙動を明確にすること。
- **再現実性**: シミュレーション結果を再現できるよう、乱数シードの設定等の機能を持たせること。

## 5. 今後の開発フェーズ
1. **要件定義 (Requirements Definition)**: 本書の合意（イマココ）
2. **基本・詳細設計 (Architecture & Interface Design)**: クラス図およびモジュール間データ仕様の策定
3. **基盤実装 (Core Implementation)**: 抽象クラスとダミー問題を用いた基盤構築
4. **オラクル・ノイズ実装 (Quantum logic Implementation)**: Groverアルゴリズム本体と複雑なノイズの適用
5. **評価と統合 (Benchmark & Integration)**: 各種グラフ出力機能と実問題でのベンチマークテスト
